# Voice and Taste

## Purpose

Preserve the writer's taste. This file is a fillable taste profile plus
instructions for using it: the writer records what they like and dislike in
prose, and any AI assistant reads it before giving feedback or (when asked)
drafting sample prose — so suggestions bend toward the writer's voice instead
of the model's defaults.

The core rule: **describe style via craft traits, never by naming living
authors to imitate.** "Short declarative sentences, dry humor, concrete nouns,
no similes in action scenes" is a usable instruction; "write like [living
author]" is not — it is imprecise, it flattens the writer's own voice, and
this workshop does not do it.

## Use when

- Setting up a new story project (fill the profile early, revise it often).
- Any feedback or revision session — read the profile first, apply it.
- The writer says "that suggestion doesn't sound like me."
- Before drafting any sample prose on request.
- The writer wants help articulating WHY they like or dislike a passage.

## Inputs to read first

- The taste profile below (or the story-local copy, if the writer keeps one
  in the story folder — e.g. a `prompts/` note).
- `story_bible.md` — tone and "things this story should avoid."
- A page or two of the writer's own recent prose from `drafts/`, which is the
  ground truth the profile approximates.

## Process

1. If the profile below is empty, help fill it: ask for a passage the writer
   loves (theirs or published), and one that grates. Pull craft traits out of
   each — sentence length, concreteness, irony level, how much is left unsaid.
2. When the writer names an author they admire, translate: identify two or
   three craft traits that make that author's prose work, record the traits,
   and drop the name. Deceased authors may be discussed as craft references,
   but the recorded profile still uses traits, not names.
3. When giving feedback, check suggestions against the profile: never suggest
   a banned phrase, respect the humor/darkness/sincerity/weirdness dials, and
   flag (rather than "fix") deliberate style choices.
4. When drafting sample prose on explicit request, follow the profile and say
   which traits you leaned on, so the writer can correct the dials.
5. Revisit the profile whenever the writer overrides a suggestion for taste
   reasons — that override is new profile data; offer to record it.

## Output format

The operator-fillable taste profile. Copy it here (or into a story-local
file) and fill it in plain language; every section may start rough and grow.

```markdown
### Prose I like
(paste or describe 1-3 passages and what works about them)

### Prose I dislike
(paste or describe passages that grate, and why)

### Preferred sentence feel
(long/short mix, plain or ornate, comma appetite, fragment tolerance)

### Humor level
(none / dry / warm / absurd — and where humor is off-limits)

### Darkness level
(how dark the work may go, and what stays off the page)

### Sincerity level
(earnest vs ironic; when the story is allowed to be fully sincere)

### Weirdness level
(how strange images and events may get before they need grounding)

### Banned phrases
(words and constructions never to suggest; clichés included)

### Recurring images I like
(motifs, textures, weather, objects that keep earning their place)

### Style traits
(craft traits only — rhythm, concreteness, restraint, POV habits.
Never the names of living authors to imitate.)
```

## Things to avoid

- Naming living authors as style targets, ever — even if the writer does
  first. Translate the name into craft traits instead.
- Treating the profile as fixed. Taste sharpens with practice; the file
  should be edited often.
- Enforcing the profile against the writer. If they break their own rules on
  purpose, that is a decision, not an error — at most, ask once.
- Averaging the writer toward generic "good prose." The point of the file is
  to protect what is particular.
- Filling the profile FOR the writer from your own defaults. Extract it from
  their reactions and their pages.

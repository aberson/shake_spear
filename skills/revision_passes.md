# Revision Passes

## Purpose

Revise one dimension of a draft at a time. A draft improves faster through several
small, focused passes than through one giant rewrite — and the writer stays in
control of what changes. Identify the target pass, stay inside it, and stop.

## Use when

- A draft exists and the writer says "revise", "polish", or "make this better".
- Quick feedback flagged an issue that deserves focused work.
- The writer wants a specific flavor shift: funnier, scarier, quieter, kid-friendly.
- A revision session (see `immersive_session.md`, revision mode).

## Inputs to read first

- The draft under revision, in `drafts/` or wherever the writer points.
- `active_state.md` — what stage the story is at; polishing too early wastes a session.
- `story_bible.md` — tone, and the "things this story should avoid" list.
- `continuity.md` — facts a revision must not quietly break.
- Any plans in `revisions/` or notes in `feedback/` already written for this draft.

## Process

1. Ask which pass the writer wants — or recommend ONE, with a reason, based on the
   draft's biggest need. Never start without a named target pass.
2. Confirm scope: which scene or section, and how much change is welcome.
3. Run only that pass. The twelve passes:
   - story logic — do events cause each other, or merely happen in order?
   - character motivation — does each choice trace back to a want, fear, or pressure?
   - emotional tension — where does the reader lean in, and where do they drift?
   - dialogue — subtext, voice, and cuts; see `dialogue_doctor.md` for depth.
   - sensory detail — replace abstractions with things a reader can see and hear.
   - pacing — scene length and paragraph rhythm; where to linger, where to cut.
   - sentence polish — rhythm, word choice, accidental echoes, clunky lines.
   - line-level compression — say it in fewer words without losing the voice.
   - humor — sharpen the jokes that exist; find the ones the draft is hiding.
   - scarier/more suspenseful version — raise dread through withholding and pacing.
   - quieter/more literary version — lower the volume; trust the reader more.
   - kid-friendly clarity — simpler sentences, concrete images, safe stakes.
4. Propose changes as suggestions, showing the original line beside each revised line.
5. Save results as a new file in `revisions/` — never overwrite the draft itself.
6. Note anything the pass surfaced for OTHER passes. Park it; do not chase it.

## Output format

```text
Pass:
Scope:
What this pass found:
Suggested changes (original -> revised):
Parked for other passes:
Next pass recommendation:
```

## Things to avoid

- Rewriting everything at once. One pass, one dimension — unless explicitly asked.
- Drifting mid-pass ("while fixing dialogue I also changed the plot").
- Flattening the writer's voice into generic smoothness.
- Destructive edits. New revision files; the original draft stays intact.
- Endlessly polishing a scene the story might still cut. Check `active_state.md` first.

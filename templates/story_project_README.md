# {{title}}

This folder is one story project (`{{slug}}`) inside the shake_spear writing
workshop. Everything about this story lives here: story bible, active state,
scenes, characters, world notes, drafts, session logs, feedback, and revision
plans. Shared AI skills and templates live at the workshop root, two levels up.

## Working in this folder

- Open it directly: `cd projects/{{slug}}` then `code .` — the local
  `CLAUDE.md` / `AGENTS.md` give AI assistants full story context.
- Start every session by reading `active_state.md` (the "start here" file).
- Your prose goes in `drafts/` — tooling never modifies files there.

## Command cheat-sheet

Run from the workshop root with `uv run ss …` (or plain `ss` in an activated
venv). From inside this folder the project argument can be dropped — commands
detect the story from your working directory.

```bash
ss session {{slug}} --type scene --minutes 45   # start a dated session log
ss scene {{slug}} "Scene title"                 # new scene card
ss character {{slug}} "Character name"          # new character profile
ss world {{slug}} "World element name"          # new world element
ss index {{slug}}                               # regenerate index.md
ss recap {{slug}}                               # refresh the recap block in active_state.md
ss status {{slug}}                              # print current state (writes nothing)
ss daily {{slug}}                               # 15-minute freewrite session
ss export {{slug}}                              # gather drafts/ into exports/manuscript.md
```

## Folder map

| Folder / file | What goes there |
|---|---|
| `story_bible.md` | Premise, tone, main character, themes |
| `active_state.md` | Start-here file; `ss recap` rewrites only its marker block |
| `continuity.md` | Established facts, reader promises, open questions |
| `decisions.md` | What was decided and why |
| `index.md` | Derived — regenerated by `ss index`, safe to overwrite |
| `characters/` `world/` `scenes/` | One markdown file per entity |
| `drafts/` | Your prose; never auto-modified |
| `sessions/` `feedback/` `revisions/` | Session logs, feedback notes, revision plans |
| `prompts/` | Story-specific prompt files you write |
| `exports/` | Derived — `ss export` writes `manuscript.md` here |

## Tag conventions

Most story files carry a `tags: []` line in their frontmatter. Conventions:

- Freeform lowercase tags, written as a comma list inside the brackets:
  `tags: [subplot, act_one, moon_oven]`.
- Keep tags short and reusable across files — character names, plot threads,
  arcs, locations — rather than one-off phrases.
- `ss index` reads frontmatter (including tags) when regenerating `index.md`,
  so consistent tags make the index and continuity work more useful.
- There is no tag registry: invent tags freely, prune during revision passes.
